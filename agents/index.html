分子 Agent

动态 Subagent 池

按需 spawn，用完回收。核心价值是上下文隔离，不是并行。

正常状态 运行中 异常/超时
S

Spawn 约束

  • 最大并行 ≤6
  • 预算检查（token/时间）
  • 约束包注入（路径+规范+超时）
C

回收规则

  • 摘要 ≤500 字
  • 不可互相调用
  • 主 Agent 汇总结果
通信协议

双层通信架构

⚡

CLI 直连（同步）

同一 session 内直接调用。速度快，但无持久化审计。适用于 Subagent spawn/collect。

📂

Vault 文件（持久化）

通过 Vault 文件通信。可审计、可追溯、跨 session。适用于 Handoff、看板消息、事件流。

模式 载体 场景
直接调用 CLI subprocess Subagent spawn
看板消息 多智能体协作看板.md 跨Agent派活
状态广播 智能体状态/*.md 忙/闲/错误
事件日志 智能体事件.jsonl 生命周期审计
Handoff task card 文件 任务移交

路由决策三要素

1

能力匹配

任务需要什么 skill？哪个 Agent 有？能力表在 .skills/ 目录维护。

2

负载均衡

谁空闲？谁已经在忙？状态文件实时反映 busy/idle/error。

3

路径权限

任务要写什么路径？谁有权限？越权写入会被 pre-write-check 拦截。

路由决策示例：用户说"帮我写 PRD" → Claudian 判定为 M4 → 检查青霉素状态(idle) → 确认路径 10-项目/ 在青霉素权限内 → 派发任务 → 青霉素执行 → 返回 Handoff → Claudian 验收。整个过程通过看板和状态文件协调，无需人工介入中间环节。
