熔断机制

三级熔断保护

状态 阈值 触发动作
绿色（正常） 预算消耗 <60% 正常运行，无干预
黄色（警告） 预算消耗 60-100% 周报标黄，建议节约
红色（熔断） 预算消耗 >100% 禁止 spawn 新子任务，仅允许收尾

熔断后恢复：等待下一周期重置，或管理员手动放行（更新 cost-fuse.json）

事件流

三阶段成本事件流

从「收工单次写入」进化为「持续三阶段追踪」，增量 delta 语义

cost_start — 任务启动时写入

{
  "ts": "2026-05-31T10:00:00+08:00",
  "event": "cost_start",
  "agent": "qingmeisu",
  "task_id": "T-20260531-01",
  "tokens": 0,
  "cost_cny": 0
}

不计入汇总，仅标记任务开始时间

cost_checkpoint — 关键阶段完成时写入增量

{
  "ts": "2026-05-31T10:15:00+08:00",
  "event": "cost_checkpoint",
  "agent": "qingmeisu",
  "task_id": "T-20260531-01",
  "tokens": 32000,
  "cost_cny": 0.45
}

计入汇总，tokens/cost_cny 为本阶段增量

cost_finalize — 任务结束时写入最后增量

{
  "ts": "2026-05-31T10:28:00+08:00",
  "event": "cost_finalize",
  "agent": "qingmeisu",
  "task_id": "T-20260531-01",
  "tokens": 18000,
  "cost_cny": 0.25
}

计入汇总，标记任务完成

事件管道流程

  1. v8_handshake 自动写入 cost_start
  2. 执行中可按需写入 cost_checkpoint（可选，多次）
  3. v8_end 自动写入 cost_finalize
  4. 事件写入 02-项目管理/cost-events.jsonl
  5. 每周 v8-cost-observability.py 生成周报
  6. 异常检测 → 触发 cost-fuse 熔断

口径规则：每次写入的 tokens/cost_cny 是增量(delta)，不是累计。汇总时对 checkpoint + finalize 求和即为任务总成本。

调用脚本

bash .standards/hooks/cost-event.sh <phase> <task_id> <agent_id> [tokens] [cost_cny]

phase: cost_start | cost_checkpoint | cost_finalize

周度降级策略

当预算紧张时的自动降级措施

消耗比例 模型策略 行为调整
<60% 正常使用 Claude Opus/Sonnet 无限制
60-80% 优先 Sonnet，减少 Opus 减少并行
80-100% 仅 Sonnet + DeepSeek 单任务串行
>100% DeepSeek only 仅允许收尾，禁 spawn

模型选用指南

场景 推荐模型 理由
PRD/方案 Claude Opus 需要深度推理
日常对话 Claude Sonnet 性价比
批量处理 DeepSeek 成本极低
向量索引 bge-m3 本地运行，零成本
