六步流程

从拆分到验收的完整管道

1. 拆分（Decompose）

把大任务拆成可独立执行的子任务，每个子任务有明确输入/输出。拆分粒度以"一个 Agent 在一个 session 内可完成"为基准。每个子任务须有：任务名、输入物、输出物、预估时间、执行 Agent。

2. 计划（Plan）

确定执行顺序、依赖关系、预估时间，选择执行 Agent。产出物：task card（M4/M5）或计划声明（M3）。明确 blockers 和并行机会。

3. 执行（Execute）

按声明写入文件，遵循档位规格执行。每次写入前自问："我声明过要写这个文件吗？"没有则先暂停、补充声明、再继续。执行中发现范围扩大须升档。

4. 监控（Monitor）

V9.2: heartbeat-update.sh 自动更新心跳（需设置 V8_AGENT_PID），heartbeat-check.sh 三级超时检测(15min 警告/30min 过期/60min 死亡+自动 idle)。M5 长任务每 30min 必须有 heartbeat，超时触发告警。原三阶段成本事件流已于 2026-07-19 退役。

5. 检核（Verify）

自检三件套：路径合规 / frontmatter 完整 / 无装饰性 emoji。Gate 门禁自动验证（pre-write-check + post-write-check）。检核不通过则阻止提交，必须修正后重跑。

6. 验收（Accept）

验收方确认产物质量。核心原则：submitted 不等于 accepted。未经验收方确认，不得声称"已验收"。验收通过后状态标为 completed，写收工三件套。
