L0 浏览者 · 2 分钟

目标：能打开 Vault 浏览文档和看板

部署进度 0/3
Step 1 详细操作：安装 Obsidian

brew install --cask obsidian
或直接下载：obsidian.md/download → DMG → 拖到 Applications

winget install Obsidian.Obsidian
或直接下载：obsidian.md/download → exe 安装包 → 双击安装

Step 2 详细操作：获取 Vault

获取方式：U盘/网盘 zip（推荐最快）、AirDrop（Mac）、或 Git clone

放到：~/Desktop/obsidianVault/

放到：C:\Users\你的用户名\Desktop\obsidianVault\

Step 3 详细操作：打开 Vault
  1. 启动 Obsidian → "打开文件夹作为仓库" → 选择 Vault 目录
  2. 等待 30 秒索引加载
  3. 按 Cmd + O 搜索「Home」→ 打开 00-MOC/Home.md
  4. 按 Ctrl + O 搜索「Home」→ 打开 00-MOC/Home.md

L1 问答者 · 10 分钟

目标：能在终端与 AI 对话，偶尔问问题

部署进度 0/5
Step 1 详细操作：安装基础环境（Node.js）

brew install node
验证：node --version 输出 v18+ 即可

winget install OpenJS.NodeJS.LTS
验证：打开 PowerShell → node --version 输出 v18+ 即可

Step 2 详细操作：安装 AI 对话工具

推荐 Claude Code（官方 CLI）：

npm install -g @anthropic-ai/claude-code

或 OpenClaw（开源替代）：

npm install -g openclaw

Step 3 详细操作：配置 API Key

Claude Code 首次启动自动引导登录，按提示完成即可。

如使用 API Key 模式：

export ANTHROPIC_API_KEY="sk-ant-..."
永久生效：追加到 ~/.zshrc

系统设置 → 环境变量 → 新建用户变量
变量名：ANTHROPIC_API_KEY，值：sk-ant-...

Step 4 详细操作：创建最小项目记忆

在 Vault 根目录创建 project-context.md：

# 项目上下文

## 这是什么
这是一个 Obsidian 知识库，用于团队协作管理。

## 目录结构
- 00-MOC/：导航和索引
- 01-项目/：项目文档
- 02-项目管理/：看板和状态

## 常用操作
- 找文件：搜索文件名关键词
- 看进度：打开「多智能体协作看板」
Step 5 详细操作：验证对话

打开终端（Terminal），cd 到 Vault 目录：

打开 PowerShell，cd 到 Vault 目录：

cd ~/Desktop/obsidianVault

claude "这个项目是做什么的？"

AI 应能基于 project-context.md 回答项目概况。

L2 产出者 · 20 分钟

目标：能让 AI 按模板产出文档，理解 V9 二元触发器

部署进度 0/9
Step 1-5：完成 L1 部署

请先切换到 L1 标签完成基础环境部署。已有 Node.js + AI 工具 + project-context.md 即可跳过。

Step 6 详细操作：配置角色设定（.claude/CLAUDE.md）

在 Vault 根目录创建 .claude/CLAUDE.md：

# 项目指令

## 身份
你是一位[角色]。

## 产出规则
- 先列骨架大纲，确认后再展开
- 不确定的标注「待确认」
- 中文，Markdown 格式

## 合规（V9 二元触发器）
写文件前要声明：V9 写入声明：{路径} | {动作摘要}
不写文件直接回复，无合规开销。

把 [角色] 替换为你的实际角色，如「产品经理」「前端工程师」等。

Step 7 详细操作：确认产出模板

浏览 30-规范/ 目录，确认已有的模板：

  • PRD 模板：产品需求文档格式
  • 技术方案模板：方案设计格式
  • 会议纪要模板：记录格式

如需自定义，在 .claude/CLAUDE.md 中添加产出格式说明。

Step 8 详细操作：验证完整工作流

测试一个完整产出流程：

  1. 进入 Vault 目录，启动 claude
  2. 下达产出指令：「帮我写一份 XX 的 PRD 初稿」
  3. AI 应先输出写入声明，再产出内容
  4. 确认产物写入位置和格式正确
Step 9 详细操作：理解分步产出法

V9 产出方法论核心：

  1. 先聊后做：用 1-2 轮对话对齐需求，不急着产出
  2. 骨架优先：先列大纲/结构，确认后再展开细节
  3. 声明后写：确认写入范围后再落盘
  4. 验后收工：产出后自查格式、路径、内容合规

V9 要点

L2 只需记住一条：写文件前要声明，不写文件直接做。

L3 协作者 · 1 小时

目标：完整多 Agent 协作环境，带门禁、状态流和成本追踪

提示：L3 是完全体，通常只有 1-2 人需要（系统管理员/架构师）。大部分团队成员停在 L2 即可。

部署进度 0/14
Step 1-9：完成 L2 部署

请先完成 L2 所有步骤。L3 在 L2 基础上增加多 Agent 协作能力。

Step 10 详细操作：安装完整工具链

安装 Python 3.12+：brew install python@3.12

安装 Python 3.12+：winget install Python.Python.3.12

安装 .standards/ 依赖：

cd ~/Desktop/obsidianVault
pip install jsonschema pyyaml

验证：python3 .standards/pre-start-check.py 无报错

Step 11 详细操作：配置多 Agent 环境

V9 支持 6 个 Agent 角色协作。配置步骤：

  1. 确认 .prompt-src/ 目录存在（源 prompt 模板）
  2. 运行 prompt 构建：python3 .prompt-src/prompt-build.py
  3. 确认 .claude/agents/ 下生成了各 Agent 设定文件

6 个 Agent：东风（架构）、红梅素（执行）、青霉素（质检）、小虫（巡检）、投保（协调）、WorkBuddy（日常）

Step 12 详细操作：启动状态表和事件流

确认以下文件存在：

  • 02-项目管理/智能体状态/*.md — 每个 Agent 一张状态卡
  • 02-项目管理/智能体状态/智能体事件.jsonl — 事件流
  • 02-项目管理/运行日志/ — 每日日志目录

初始化状态：所有 Agent 的 status 应为 idle。

Step 13 详细操作：配置 Gate 门禁

Gate 门禁确保 Agent 不越权写入。配置方式：

  1. 确认 .standards/hooks/pre-write-hook.sh 存在（v1.1+）
  2. 在 .claude/settings.json 中配置 hook： 
    {
  "hooks": {
    "PreToolUse": [{
      "matcher": "Write|Edit|NotebookEdit",
      "hook": "bash .standards/hooks/pre-write-hook.sh"
    }]
  }
}
  3. 设置环境变量（多 Agent 共用 hook 时）：V8_AGENT_ID=claudian
  4. 确认 .standards/agent-contract.yaml 存在（V9.2 必需）
  5. 测试：让 AI 尝试写入未声明路径，应被拦截
Step 14 详细操作：验证完整协作流

完整验证清单：

  1. 启动一个 M4 任务，确认状态文件更新
  2. 确认事件流 .jsonl 有 task_start 记录
  3. 完成任务，确认收工三件套输出
  4. 检查看板是否自动更新
  5. 确认成本事件已追踪
